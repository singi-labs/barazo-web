## Design Context

### Users

**Primary:** Community builders and forum administrators who want to run discussion spaces without surveillance, vendor lock-in, or losing their members' data. Technical enough to self-host (or choose managed hosting), but not necessarily developers.

**Secondary:** Forum members — people who participate in discussions, post content, and build reputation across communities. Range from casual visitors to power users. Many arrive from the Bluesky/AT Protocol ecosystem with existing identities.

**Context of use:** Desktop-heavy for long-form discussion reading/writing, but mobile for quick replies and notifications. Dark mode default (forum culture leans dark), system preference respected. Used in focused reading sessions — readability and information density matter more than visual flair.

**Jobs to be done:**
- Run a community forum with portable identity (AT Protocol)
- Participate in discussions without creating yet another account
- Build cross-community reputation that travels with your identity
- Self-host community infrastructure with full data ownership
- Administer and moderate communities effectively

### Brand Personality

**Three words:** Dependable, straightforward, community-owned.

**Voice:** Knowledgeable friend who happens to be an expert. Technical credibility with warmth. Not corporate, not overly casual. Direct and honest — if there's a limitation, say so.

**Emotional goals:** Trust (in infrastructure ownership), belonging (community-first, not platform-first), competence (admin tools that respect your intelligence).

**Anti-references:**
- NOT Reddit (ad-supported, closed, opaque moderation)
- NOT Discord (ephemeral, unsearchable, surveillance-heavy)
- NOT generic SaaS dashboards (hero metrics, gradient accents, glassmorphism)
- NOT Web3/crypto aesthetic

**References:** Discourse (functional density, trust levels — but modernized), well-designed developer tools, mature open source project UIs that prioritize usability over aesthetics.

### Aesthetic Direction

**Visual tone:** Functional, warm, readable. Sand-toned neutrals create warmth without being decorative. Cyan accent used sparingly for interactive elements. Information-dense forum layouts that breathe through spacing, not decoration.

**Theme:** Dark mode default (forum community convention), system preference enabled. Both themes are first-class — dark is not "to look cool," it's because forum users read long threads and prefer reduced eye strain.

**Typography note:** Currently uses system font stack for body text. **This is an active gap** — no custom display or body font has been chosen yet. When selecting fonts for Barazo, Impeccable's font selection procedure should be followed in full. The font should complement the warm, dependable personality and support long-form reading. Do NOT default to the same fonts as Sifa (iA Writer Quattro / Space Grotesk) — each product should have its own typographic identity.

### Design Principles

1. **Content is king** — Forum UI serves the discussion. Thread readability, clear hierarchy between posts/replies, scannable metadata. UI chrome should recede.
2. **Functional density** — Power users need information density. Progressive disclosure for complexity, but don't hide things behind too many clicks. Admin panels can be dense.
3. **Warm neutrals, cool accents** — Sand (Radix) for warmth and readability, Cyan (Flexoki) for interaction, Purple for secondary/info. The palette should feel inviting, not clinical.
4. **Self-hosting is a first-class experience** — UI must work beautifully on small VPS deployments. No features that assume cloud scale. Docker Compose is the deployment model.
5. **Accessibility from first commit** — WCAG 2.2 AA minimum. Forum content must be readable by everyone. Semantic HTML for screen readers navigating long threads.

## Design System (Established — Do Not Override)

### Fonts

| Role | Font | Notes |
|------|------|-------|
| Body/UI | System font stack | `-apple-system, BlinkMacSystemFont, Segoe UI, Roboto, sans-serif`. **No custom body font yet — this is an intentional gap awaiting font selection.** |
| Code | Source Code Pro | Via `next/font/google`, weights 400-700, `font-display: swap` |
| Display | None yet | **Open for selection.** Follow Impeccable's font selection procedure when choosing. |

### Colors (INTENTIONAL — preserve this system)

**Architecture:** Radix Colors (12-step scales, automatic light/dark) + Flexoki accent hues.

- **Neutrals:** Sand scale (warm) + Gray scale (cool neutral)
- **Primary accent:** Cyan (Flexoki-inspired, `--cyan-9`)
- **Secondary:** Purple (`--purple-9`)
- **Semantic:** Green (success), Orange (warning), Red (error) — all Radix 12-step scales
- **Focus ring:** `--cyan-8`

**Token system:** All colors via CSS custom properties mapped through `@theme inline`. Radix dark scales auto-switch in `.dark` class. No hardcoded hex in components (except two contrast-critical foreground values documented in globals.css).

### Spacing & Radius

- Border radius: sm (0.25rem), md (0.375rem), lg (0.5rem), xl (0.75rem), 2xl (1rem), full (9999px)
- Shadows: xs through xl scale
- Container: `max-width: 80rem`, `padding-inline: 1rem`, `margin-inline: auto`

### Icons

Phosphor Icons exclusively (`@phosphor-icons/react`).

### Component Library

shadcn/ui + Radix UI primitives. Tailwind utility classes only — no inline styles.

### Motion

- `prefers-reduced-motion` fully respected (all animations/transitions set to 0.01ms)
- Existing animations: slide-in/out-left, fade-in/out (200ms, ease-out/ease-in)
- Selection color uses primary-muted/primary tokens

### Responsive

Mobile-first. Tailwind breakpoints. Touch targets 44x44px minimum.

### Accessibility

- WCAG 2.2 AA minimum
- `eslint-plugin-jsx-a11y` strict mode
- `vitest-axe` + `@axe-core/playwright` + `pa11y-ci` in CI
- Focus visible: 2px solid ring with 2px offset

## Anti-Pattern Overrides

1. **Dark mode IS the default** (`defaultTheme="dark"`) — this is forum community convention for long reading sessions, not "defaulting to dark to look cool." System preference is also enabled.
2. **Cyan accent is primary** — yes, Impeccable warns against "AI color palette: cyan-on-dark." Our cyan comes from Flexoki and is used sparingly (10% rule) for interactive elements only, not as an ambient glow. Warm Sand neutrals dominate.
3. **Card patterns used** for forum posts and thread containers — semantic containers, never nested. Post cards serve as content boundaries in thread view.

## Where Impeccable Should Add Value

1. **Font selection** — When choosing display/body fonts for Barazo, follow Impeccable's full font selection procedure. This is the biggest design gap right now.
2. **Component polish** — `/polish` and `/audit` on existing UI components to catch spacing inconsistencies, contrast issues, and interaction state gaps.
3. **Anti-pattern detection** — Run CLI on existing JSX to catch any AI tells that crept in during rapid development.
